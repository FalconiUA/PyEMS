Device simulation

The simulator runs the site's device profiles as real Modbus TCP servers (per config/site.sim.yaml) so the EMS can be tested — and broken — without hardware. Drive PV generation and load by hand, with a synthetic curve, or by replaying recorded 1-second CSV data, and inject faults: device offline, frozen registers, Modbus exceptions, rejected writes, ignored setpoints.

The EMS itself is started separately, exactly like in production: pyems --site config/site.sim.yaml
